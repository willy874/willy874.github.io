## 0. 專案定位

「竹子日誌」是以 **竹子** 為筆名經營的技術部落格,內容以繁體中文為主,涵蓋:

- **Zero-Trust / OpenZiti 系列**:入門 HTML 解說頁、技術深度演講、SDK 整合實作
- **前端工程**:React / TypeScript / TanStack / Ant Design v6 / 設計系統
- **AI 工作流**:Claude Code、subagent 架構、MCP 整合實務
- **演講與簡報**:conference talks(如「來造一個下拉選單的輪子」)的簡報 HTML 存檔
- **互動 Demo**:如紫微斗數命盤視覺化等獨立 HTML 工具頁

因此本站不只是「文章列表」,而是 **文章 + 簡報存檔 + 互動工具頁 + 靜態檔案庫** 四種內容形態並存的個人技術站。

## 2. 技術選型與理由

### 2.1 核心框架:Astro 5.x

選擇理由(相對於 Next.js / Hexo / Jekyll):

1. **Islands Architecture**:預設輸出純 HTML、零 client JS。技術 blog 90% 內容是靜態文字,只有少數互動 demo 需要 hydration,Astro 的 `client:*` 指令精準控制這件事。
2. **React 相容**:透過 `@astrojs/react` 直接沿用既有 React/TS 技能與元件習慣,不需學新的模板語言(對比 Hugo 的 Go template、Jekyll 的 Liquid)。
3. **Content Collections**:frontmatter 有 Zod schema 驗證,type-safe,與平常使用 zod + openapi-zod-client 的習慣一致。
4. **原生支援混合路由**:`.astro`、`.mdx`、`.html` 都能當 page,簡報 HTML 直接丟進 `src/pages/` 就有路由,完美符合 R2/R5。

### 2.2 明確「不採用」清單

| 技術 | 不採用理由 |
|------|-----------|
| Ant Design | Blog 不是後台系統;antd bundle 對靜態站過重,且 SSR/SSG 下 CSS-in-JS(v5)或 CSS Variables(v6)都是不必要的複雜度。互動 demo 若需要再以 island 局部引入 |
| TanStack Query | 無 runtime API 請求需求;資料全部 build-time 取得 |
| Next.js | GitHub Pages 只能 `output: 'export'`,會閹割 Image Optimization / ISR 等主要優勢,等於用殘缺的 Next 換更重的心智負擔 |
| CMS(Tina/Decap) | 單人 blog,Git + Markdown + Claude Code 即是最好的 CMS |

### 2.3 完整技術清單

| 層次 | 技術 | 版本 | 備註 |
|------|------|------|------|
| 框架 | Astro | 5.x | `output: 'static'` |
| UI islands | React | 19.x | 只用於互動 demo |
| 語言 | TypeScript | 5.x | `strict: true` |
| 樣式 | Tailwind CSS | 4.x | CSS-first config,搭配 CSS variables 做主題 |
| 內容 | MDX | — | `@astrojs/mdx` |

| 搜尋 | Pagefind | 1.x | **支援 CJK 斷詞**,詳見 §7.1 |
| 評論 | giscus | — | GitHub Discussions 驅動,支援 zh-TW 介面 |
| Sitemap | `@astrojs/sitemap` | — | |

## 3. Repo 與網址策略

### 3.1 Repo 命名(二選一)

| 方案 | Repo 名稱 | 網址 | base 設定 |
|------|----------|------|----------|
| **A(建議)** | `{username}.github.io` | `https://{username}.github.io/` | `base: '/'`,路徑最乾淨 |
| B | `bamboo-blog` | `https://{username}.github.io/bamboo-blog/` | `base: '/bamboo-blog'`,所有內部連結都要處理前綴 |

**建議採方案 A**:簡報連結會被分享到社群與投影片 QR code,`/talks/zerotrust-2026` 遠比 `/bamboo-blog/talks/zerotrust-2026` 好記好讀。未來若購買自訂網域(如 `bamboo.blog`)也只需加 CNAME,路徑結構不變。

### 3.2 URL 設計規約

```
/                          首頁(近期文章 + 精選演講)
/posts/                    文章列表(支援 tag 篩選)
/posts/{slug}/             文章內頁
/posts/tags/{tag}/         Tag 彙整頁
/talks/                    演講列表(卡片含日期、場合、簡報連結)
/talks/{slug}              簡報 HTML 本體(獨立全螢幕頁,無站台 layout)
/tools/{slug}              互動工具頁(如紫微命盤)
/about/                    關於竹子
/files/**                  純靜態檔案(PDF、圖片、附件),不進 sitemap
/rss.xml                   RSS 全文輸出
/404                       自訂 404
```

命名規則:

- slug 一律使用 **英文 kebab-case**(如 `zerotrust-dark-services`),避免中文 URL encode 後難以分享
- 檔案類資源放 `/files/{年份}/`,如 `/files/2026/zerotrust-slides.pdf`,方便日後清理

### 4.1 靜態檔案曝露的三條路徑(R2 詳解)

| 用途 | 放置位置 | 輸出行為 | 範例網址 |
|------|---------|---------|---------|
| PDF、附件、原始圖 | `public/files/**` | 原封不動 copy | `/files/2026/zerotrust-slides.pdf` |
| 簡報 HTML(要乾淨路由) | `src/pages/talks/*.html` | 直接成為 page 路由 | `/talks/zerotrust-2026` |
| 互動工具(要主題/元件) | `src/pages/tools/*.astro` | 走 Astro 編譯 | `/tools/ziwei` |

**注意事項:**

- `src/pages/` 下的 `.html` 會被 Astro 當作 page 處理但**不會注入任何東西**,自帶 `<style>`/`<script>` 的簡報檔可直接使用——這正是既有 zero-trust 簡報 HTML 的理想歸宿。
- 若簡報 HTML 引用相對路徑資源(圖片、字型),資源放 `public/talks-assets/{slug}/`,並在簡報內用絕對路徑 `/talks-assets/{slug}/...` 引用,避免路由層級變動時斷鏈。

### 4.2 GitHub Pages 硬限制(檔案庫必讀)

| 限制 | 數值 | 因應 |
|------|------|------|
| 單一檔案 | < 100 MB(git 限制) | 大型影片改嵌 YouTube;**Git LFS 檔案 Pages 不會 serve,禁用** |
| 站台總容量 | 建議 < 1 GB | `/files/` 按年份歸檔,定期清理 |
| 頻寬 | 軟限制 100 GB/月 | 個人 blog 遠低於此,無虞 |

## 5. Content Collections Schema

`src/content/config.ts`:

```ts
import { defineCollection, z } from 'astro:content';

const posts = defineCollection({
  type: 'content',
  schema: ({ image }) =>
    z.object({
      title: z.string().max(60),
      description: z.string().max(160),        // 兼作 meta description
      publishDate: z.coerce.date(),
      updatedDate: z.coerce.date().optional(),
      tags: z.array(z.string()).default([]),
      series: z.string().optional(),           // 如 "zero-trust"
      cover: image().optional(),               // 走 astro:assets 最佳化
      draft: z.boolean().default(false),
      lang: z.enum(['zh-Hant', 'en']).default('zh-Hant'),
    }),
});

const talks = defineCollection({
  type: 'data',                                // YAML metadata
  schema: z.object({
    title: z.string(),
    event: z.string(),                         // 場合,如 "JSDC 2026"
    date: z.coerce.date(),
    slideUrl: z.string(),                      // /talks/zerotrust-2026
    pdfUrl: z.string().optional(),             // /files/2026/....pdf
    videoUrl: z.string().url().optional(),
    abstract: z.string(),
  }),
});

export const collections = { posts, talks };
```

設計要點:

- **`series` 欄位**:zero-trust 系列文可在文章頁自動列出同系列前後篇,取代手動維護連結。
- **`talks` 用 data collection**:簡報本體是 HTML 檔,metadata 抽離成 YAML,`/talks/` 列表頁純粹讀資料渲染,關注點分離。
- **`draft: true`**:build 時過濾,可以安心把未完成稿 push 上 main。

## 6. 設計系統與中文排版

### 6.1 字型策略

```css
/* global.css — Tailwind v4 @theme */
@theme {
  --font-sans: "Noto Sans TC", ui-sans-serif, system-ui, sans-serif;
  --font-mono: "JetBrains Mono", "Noto Sans TC", ui-monospace, monospace;
}
```

- **正文**:Noto Sans TC(Google Fonts,`font-display: swap`,只載 400/500/700 三個字重 + `unicode-range` 子集化)
- **程式碼**:JetBrains Mono,中文 fallback 到 Noto Sans TC 避免中英混排時中文變系統字
- 標題不另用 serif:繁中網頁載兩套 CJK 字型代價太高(每套 1MB+),用字重與字距做層次即可

### 6.2 中文排版細節

| 項目 | 設定 |
|------|------|
| 行高 | 正文 `leading-[1.9]`(繁中閱讀最適區間 1.8–2.0) |
| 段寬 | `max-w-[68ch]` 約 34 個全形字 |
| 標點 | `text-wrap: pretty` + `hanging-punctuation`(漸進增強) |
| 中英混排 | 交由內文書寫規範處理(中英間手動空格),不引入額外 JS |
| lang 屬性 | `<html lang="zh-Hant-TW">`,影響瀏覽器字型選擇與斷行 |

### 6.3 Dark Mode

- CSS variables 定義雙主題色票,`data-theme` 切換,`prefers-color-scheme` 為預設值
- 切換 script 以 inline `<script is:inline>` 置於 `<head>` 最前,避免 FOUC
- Shiki 雙主題輸出(`themes: { light, dark }`),程式碼區塊隨主題無縫切換

## 7. 站台功能細節

### 7.1 全文搜尋:Pagefind(CJK 重點)

Pagefind 對中文內容需要明確設定,否則斷詞會失效:

1. `<html lang="zh-Hant-TW">` 必須正確——Pagefind 依 lang 屬性選擇斷詞器
2. build script 串接:`astro build && pagefind --site dist`
3. 搜尋 UI 掛在 `/posts/` 列表頁與 Header 的 `⌘K` modal(React island,`client:idle`)

### 7.2 評論:giscus

- 需開啟 repo Discussions,giscus app 安裝後以 `data-lang="zh-TW"` 載入
- 主題跟隨站台 dark mode(監聽 `data-theme` 後 postMessage 切換)
- 以 `client:visible` 延遲載入,不影響 LCP

### 7.3 RSS / SEO / OG

- RSS 全文輸出(`content` 欄位帶入 rendered HTML),對訂閱者友善
- 每頁 canonical、`og:*`、`twitter:card` 由 `Base.astro` 統一輸出
- OG image:Phase 2 以 satori 系方案 build-time 產生,需自備 Noto Sans TC 字型檔(subset 後嵌入),否則中文標題會變豆腐

### 7.4 無障礙(WCAG 2.1 AA 基準)

- 色彩對比 ≥ 4.5:1(雙主題皆驗證)
- 鍵盤焦點可見(`focus-visible` ring)
- `prefers-reduced-motion` 尊重(所有 transition 包在 media query 內)
- 簡報 HTML 頁為例外區(第三方工具產出),於 `/talks/` 列表註明

## 8. CI/CD:GitHub Actions

`.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:        # 手動觸發(改 giscus 設定等不動內容時用)

concurrency:
  group: pages
  cancel-in-progress: true  # 連續 push 只跑最後一次

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm

      - run: pnpm install --frozen-lockfile
      - run: pnpm build          # astro build && pagefind --site dist

      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

`astro.config.mjs` 關鍵設定:

```js
import { defineConfig } from 'astro/config';
import react from '@astrojs/react';
import mdx from '@astrojs/mdx';
import sitemap from '@astrojs/sitemap';

export default defineConfig({
  site: 'https://{username}.github.io',
  base: '/',
  output: 'static',
  integrations: [react(), mdx(), sitemap()],
  markdown: {
    shikiConfig: {
      themes: { light: 'github-light', dark: 'one-dark-pro' },
    },
  },
});
```

發文流程:**寫 MDX → push main → 約 1–2 分鐘後上線**。無 preview 環境需求(本地 `pnpm dev` 即所見即所得)。

## 9. Claude Code 工作流整合

既有 Claude Code 重度使用習慣可直接套用到 blog 維運:

### 9.1 `CLAUDE.md` 專案指引

```markdown
# 竹子日誌

- 文章位於 src/content/posts/,格式 MDX,frontmatter schema 見 src/content/config.ts
- 語言:繁體中文(台灣用語),筆名「竹子」,語氣輕鬆但技術精確
- 中英混排:中文與英文/數字間加半形空格

- 新增演講:HTML 放 src/pages/talks/,metadata 放 src/content/talks/*.yaml
- commit 前執行 pnpm build 確認 schema 驗證通過
```

### 9.2 建議 Skill:`new-post`

建立一個 scaffold skill,輸入標題與 tags 即產生:

- `src/content/posts/{slug}.mdx` 含完整 frontmatter
- 封面圖 placeholder 於 `src/assets/covers/`
- 系列文自動帶入 `series` 欄位

### 9.3 自動化(Phase 3+,選用)

- Telegram MCP:每次 deploy 成功後推播文章連結到自己的頻道
- 排程 workflow:每月跑 link checker(`lychee`)檢查外部連結失效
