最近上了 Claude Max 之後，完全改變我對於很多東西的使用習慣和使用態度，它帶來的便利真的是無以倫比。最重要的我覺得是把 AI 跟生活掛勾，只有這樣才會開始用 AI 來思考事情。

我覺得最近可以分享一些有趣的生活生產力用法。

故事一

我一直是使用 IOS 備忘錄，非常喜歡它的雲端同步功能，我一用用了快要十年。裡面累積的文章數量非常之龐大，總文字量高達 1GB。隨著我成為軟體開發者，我發現 git, markdonw, claude code 這些工具，這個備忘錄沒辦法成為我管理文章和資料的群集，開始想轉移，但文章數實在太多了，手動轉移真的很不現實。
AI 時代，我呼叫了 Claude Code 幫我搜尋與分析，我發現 IOS 備忘錄其實存在電腦的一個 SQL Lite 之中，我請 Claude 幫我寫一份腳本處理。我直接把資料撈取出來，再轉換成 Markdonw，改用 Obsidian 來同步，然後用 Claude 直接幫我分類與處理無用資料，敏感資料撈出來另外存放，最後用 git 版本控制。
原本巨困難的任務突然變的非常簡單，這是非常方便的生產力工作。

故事二

因為我個人蒐藏的文字量實在過於龐大，不管是要搜尋還是批量修改都會無比麻煩，又不能單純用腳本處理，因為有時候有很多模糊地帶，它沒有精準的做法。
這時候一樣呼叫 Claude Code ，給一個模糊的大綱做法，它能高速掃描龐大的文字，並且不是使用單純的文字搜尋，可以採用語意化的搜尋。甚至可以批量加上特定的文字或標記，管理龐大的文字。自己也可以追加一些結構化的搜尋樹和標籤，提高可被搜尋到的特徵。如此一來，很多筆記其實不用記得太過於詳細複雜，只要曾經筆記起來，就能有豐富的文字去分析和追尋，徹底做到大腦記憶外包。
筆記也不需要煩惱怎麼寫，可以請 AI 當小助手，協助分析怎麼做筆記，開出好的架構，以利下次回想。

故事三

只要有公開 API 或 MCP  的工具，就可以考慮讓 AI 介入處理，成為你個人的管理工具。
可以處理的非常漂亮。
當然，只要是 MCP 都需要有自己的 Review 機制，以免 AI 失誤或是執行出格的行為。但實際上可以節約的時間實在很嚇人，這種體感真的太爽了！
