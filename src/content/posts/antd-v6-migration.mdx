antd v6 最大的變化之一,是主題實作從 v5 的 CSS-in-JS(emotion 風格)轉向 **CSS Variables**。這篇是我在專案遷移時的筆記。

## 為什麼要遷

- **SSR/SSG 友善**:CSS Variables 不需要 runtime 產生 style tag,首屏更乾淨
- **主題切換更輕**:改變數即可,不必重算整套 token
- **與設計系統對齊**:設計端的 design token 可直接映射成 CSS variables

## 遷移重點

1. 檢視所有用到 `theme.useToken()` 的地方
2. 自訂主題改以 `ConfigProvider` 的 CSS Variables 模式提供
3. 注意第三方元件是否也依賴舊的 token 注入方式

## 小結

對**後台系統**來說 v6 值得升;但對這個 blog,我選擇完全不引入 antd —— 靜態站不需要一整套後台元件庫,互動 demo 若需要再以 island 局部引入即可(這也是規劃書 §2.2 的取捨)。
