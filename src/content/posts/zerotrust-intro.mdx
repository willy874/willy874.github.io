傳統網路安全建立在一個假設上:**內網是可信的,外網是不可信的**。於是我們在邊界架起防火牆與 VPN,只要進得了內網,幾乎就暢行無阻。

這個假設在雲原生時代徹底崩壞。

## 邊界防禦的三個破口

1. VPN 一旦被突破,攻擊者就取得「內網位置」,而位置在舊模型裡等於信任。
2. 員工、外包、SaaS、行動裝置讓「內外」界線模糊,邊界本身失去意義。
3. 橫向移動(lateral movement)幾乎不受限,一台機器淪陷等於全網淪陷。

## Zero-Trust 的核心假設

Zero-Trust 把假設反過來:**永不信任,每次都驗證(never trust, always verify)**。

- 網路位置**不**代表信任
- 每一次連線都要驗證身分、裝置狀態與授權
- 最小權限,預設拒絕

```ts
// 一個 Zero-Trust 心智模型的偽代碼
function authorize(request: Request): Decision {
  const identity = verifyIdentity(request);      // 你是誰?
  const posture = checkDevicePosture(request);   // 裝置健康嗎?
  const policy = evaluatePolicy(identity, request.resource); // 有權限嗎?
  return policy.allow && posture.ok ? 'ALLOW' : 'DENY';
}
```
