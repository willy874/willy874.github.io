我向來都會說自己是「前端工程師」，全端工程師對我來說是「貶義詞」。

但我會不會後端？

- 我能夠開 API
- 我會下 SQL Query
- 我會資料庫設計
- 我懂分層架構
- 我會物件導向設計
- 我會利用 ORM 工具
- 我會 Migration 管理資料庫
- 我會容器化部署
- 我知道如何設計可觀測系統
- 我知道怎麼用 Socket 去串接 http 之外的協定
- 我會設計微服務，如何才能橫向系統拓展
- 我知道 MQ 在微服務可以解決哪些議題
- 我知道憑證加密傳輸如何運作
- 我會權限設計，基本的如 RBAC

⋯⋯等等（再寫下去好像有點硬要寫

硬幹我還是都幹的出來，但我從來不敢說我懂，真的都是淺淺的略懂。我自豪花了很多時間學習前端，我也都把時間投資在 JavaScript 這門語言，我一直都是走專精路線。

其實有時候挺受不了有人這樣對我說：「你前端學那麼久了，該學後端變成全端工程師了！」這句話充滿著說教感，我不知道他這個是基於什麼底層邏輯，我只能說術業有專攻⋯⋯我可從來沒聽說過有人會教育：「你後端學那麼久了，該學前端了」，但不少人卻喜歡反過來教育前端。

這不就是雙標嗎？
