之前分享了從面試官角度談UIUX 設計師的履歷，身為前端工程師我也想談談這一塊，最近幫朋友看了非常大量的履歷，發現不管是 junior 還是 senior 都了類似的毛病，明明能力還不錯，卻沒辦法在履歷上呈現自己的價值。

面試官在看履歷時，你要想後面很可能有100份各種滿滿文字的履歷，也可能看了100份跟你很相似的履歷，只會用10秒鐘決定「我要不要認真看這一份履歷」，如何吸引他願意認真看？

1. 只寫重點，不要廢話。這點很重要很重要！沒有人在意你個人的事，或許寫學習歷程和轉職前的經歷有些面試官會看，但這些不會是最重要的，大家在意的事你「現在」的技能和軟體經歷。
2. 技能不要流水帳寫，要用最短的文字讓人知道你的技能強度在哪裏。如果沒寫，就當你是剛學會，只懂粗淺的使用。舉例來說，webpack: 能客製化打包路徑，熟悉處理loader套件的版本依賴問題。這樣就可以很精確的呈現實力。
3. 自我介紹一行就夠了，不要當自傳和文章寫了一大堆廢話，因為根本沒有人會認認真真的看。
4. 經歷不要寫「串接API」、「撰寫技術文件」、「依據設計稿製作畫面」、「與團隊溝通討論需求」。哪一個前端工程師不是在做這些事？當每一份履歷的每一份工作都長得差不多時，真的對這些文字已經啟動「自動忽略模式」，看完只會「沒印象」。既然要寫，請寫出這份工作跟其他前端工程師相比較特殊的部分。比方說「製作google extension 」、「為團隊導入打包工具，提高部署時效率50%，降低網站消耗流量10%」、「開發團隊共用web components」等等比較亮眼的經歷。也盡量找出「數字」來表現成果，會讓一切看起來更亮眼。
5. 除了履歷，還有呢？許多工程師做一陣子後也不再維護自己的 github，導致裡面空空如也。沒有作品不是原罪，但沒有code讓人更瞭解你，也沒有部落格，更沒有名氣，那你到底怎麼展現出你的亮眼之處？這其實都是經營個人品牌的環節，除了Github 可以撰寫 Profile Overview，也可以寫文章記錄技術經驗，更可以經營社群建立個人品牌創造影響力。
6. Github 除了給，也寫了專案，但 commit 的頻率根本寥寥無幾。加上沒有README可以導讀，讓面試官在茫茫大海在尋找亮點，這真的一點也不加分。具體就要讓人從外部連結一進 Profile 就知道要看什麼專案，打開專案就看得懂要看什麼檔案，快速引導到關鍵檔案。

網路上很多花里胡喬的履歷模板，弄一份客製化履歷放在網路上。不要樣樣靠 104 履歷，你無法脫穎而出，只會被當作炮灰而已。目前我幫許多人健檢過履歷，問題大同小異，希望大家可以越來越好！
