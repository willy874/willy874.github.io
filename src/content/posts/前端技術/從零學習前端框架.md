我一直有從事程式教學，從新手開始，有些人真的連 HTML 都不認識，一路往下學習切版、程式、框架。網路上其實一面倒政治正確說著要學習「React」才是正確的選擇，我其實也能夠理解為什麼學 react 更好，但背後有很多週邊因素。我無論如何依然是推薦新手學 vue，以下理由我會逐一說明。

1. 不需要太懂 JavaScript 就可以寫出許多邏輯複雜的操作。模板最方便的地方就是可以讓作者輕易自訂語法，許多黑魔法可以快速讓新手上手，但黑魔法特別容易讓老手感冒。
2. Vue 不會寫到特別好，但也不會寫到特別差。我要說的是 optional api ，不是指 composition api ，這兩者特性不同。如果你看過新手寫 React 就能明白，不但寫的慢，而且根本看不懂邏輯，一堆東西都攪在一起。
3. 公司教育成本低，反正完全不懂前端也沒關係，只要看得懂 html ，粗淺懂 JavaScript ，我告訴他模板怎麼寫一下就上手了，也不用花很多錢請「厲害的工程師」。這個企業方思維下其實已經很明白，就是一個成本考量，對於中小企業來說這更加明智。
4. 承如上一點先決條件，Vue工作機會豐富。我要講的並不是 React 不好找工作，我要表達的是 Junior 不好找寫 React 的工作。我身邊就有太多血淋淋的案例，就是資歷經驗不足就是找不到 React 的工作，學 Vue 的早就下職場了。
5. 常常有人說寫 React 就是在寫 JavaScript ，對於長遠來說是好事，對於修煉基本功很重要。但教學不是這樣，你要的是讓她能寫出功能建立自信比較重要，不然還沒上手就放棄了。舉例來說，迴圈你不需要分辨 array 和 object 迭代方式的差異，也不用學習多種陣列操作的差異(map, filter, concat)，更不需要關心陣列的 references 有沒有改變。以上舉例真的對新手來講太友善了。
6. 組件的依賴偵測和渲染優化，Vue 可以說是全自動化，動態處理。幾乎可以說是無差別幫你處理所有 dependencies ，也不需要對 component 去包 memo，因為框架背後做完了，效能不會90分，但不容易不及格。
7. 前面有講到，成就感很重要。快速能寫出功能遠比學好基礎有時候對於新手重要太多了，畢竟這條路很長，早點進職場總是好的。

其實真的每一點要往更細的角度談都還可以以細分列舉，不過還是留著下一篇說吧。
