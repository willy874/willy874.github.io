網路上清一色是捧 react 踩 vue，一派說法是 vue 比 react 用的人少，一派說法是 vue 是中國人寫的，還有一派說法是 vue 黑魔法多。多半我會護航 vue，因為許多人噴的都不在點上，我覺得只是框架優越感而已。但今天我要平反一下，vue 相較 react 有什麼使用上的不好用？

1. 模板語意和腳本語意不一致。常常在寫 setup 和 render 時會錯亂，常常遲疑在要不要寫 “.value” 這問題。
2. 設計 HOC、extends 非常不方便，它並無一套簡單卻能完美繼承，需要手動為 slots 加工。不像 react 直接解構賦予給 children node 上層的 props 就可以解決。
3. 當你想封裝一個功能或元件，需要為 vue 多種功能與多個生命週期進行邏輯處理。
4. 因為功能太多，寫法過於自由，生態圈要跟上相關功能的延展經常需要比較長的時間。
5. 對於 jsx 的語法支援和提示未能完全跟上 template 的便利性。對於喜歡 jsx 的人，常常無法體驗到更好的提示和效能優化。
6. 對於元件型別的定義過於複雜，學習成本過高。3.3 之後的泛型寫法對於 setup script 以外的原生寫法不支援，需要比較大的心力去定義。

我完全用「使用者」立場去看這件事，我也是 vue 的支持者，但事實的問題是我會因為我支持而不在，這不是「主觀問題」，是確實存在的問題。
