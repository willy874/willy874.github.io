一直以來我都是在寫 vue ，調過來轉換到 react 後強烈的感受到不同的思維方式。換框架坦白說真的不困難，有奠基著足夠 js 的基礎不是太大的問題，單純只為了把功能寫出來並不難，對任何框架都是如此。

但你開始思考著如何設計元件、元件的重用性、優化效率與效能⋯⋯等等進階問題，你會發現 react 排山倒海的問題冒出來，每一個都牽扯複雜的渲染機制，你要在腦海中建立一個生命週期的循環。心裡默默在想，那些該死的 deps不能再更聰明一點嗎？這memoized 難道都得自己處理嗎？

反觀 Vue 呢？你只要關心它有沒有被觸發檢查機制就行，剩下它會自己完成檢查和更新。很多東西的性能優化處理、參數的拋接和傳遞、閱讀性的一致性，等等東西其實都不用太煩惱，因為再差也就這樣。

但也不得不說，生態圈的繁榮，和對於自由的追求性，這大概就是它最大的優點吧。寫起來不會有太多拘束感，也不會有很多模板語法上的禁忌，你可以直覺的用js 去思考問題。對於效能的運算你可以選擇「處理」或「忽略」，可以對每一個環節的計算做更細緻入微的優化。

我可以說，如果你是一個工程師會很喜歡寫 react ，可以對一切極盡追求，讓程式可以更漂亮，讓效能可以更有效率運作。但如果面向市場 vue 更快更有效率，也更好學習更容易教人。
