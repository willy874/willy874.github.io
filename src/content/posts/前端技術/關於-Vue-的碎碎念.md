我很喜歡 Vue ，它自由、限制少、出錯率低、自動又聰明、官方提供超級豐富的 API 來掛鉤底層各種行為。

但正因為它超高容錯率，也導致它超級容易被寫出「很多問題」的程式碼。因為太容易學習，導致新手可以搞不清楚狀況就上手，也誕生許多非常可怕的程式碼。

網路上經常許多人形容 Vue 很多黑魔法，對我來說，那些工具本質就是 JavaScript ，沒有什麼魔法，魔法來自於「不了解」。很多人都稱之那些「神奇又方便的功能」叫魔法，我會說⋯⋯當你還當它是魔法的那天，你更應該知道它的意義是什麼，而非當它是解決你眼下問題的萬靈丹。

各種 API 是讓你開發上能做出彈性更大的設計，而不是「濫用」。我認為「業務元件」絕對用到越少的 API 越好，用最少的功能做出最豐富的設計。
