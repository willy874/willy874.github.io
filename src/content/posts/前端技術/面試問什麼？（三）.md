「公司使用哪些技術線？」

這題就是做我個人的最終篩選，其實能曬出蠻多東西的，結果上來說會很有趣。

大部分都會很籠統跟你講什麼 Vue, React 之類，但魔鬼藏在細節，如果你去追問可以發現很多東西講的不是那麼清楚。以下任何問題只要面試的用人主管任何一句「不知道」或「不清楚」就很嚴重了。或是跟你說「忘記了」、「不是我在處理」你要提高警覺去追問。

其中最常見，談到「MVC」、「jQuery」都是危險訊號。現在不使用框架要找前端工程師，這都很高機率前後端不分離，最怕一隻檔案看到 SQL Query 和 HTML，整個結構真的很噁心。

再來可以問採用什麼 UI Framework？如果回答你手刻，你就必須追問管理方式。有沒有使用 storybook？採用什麼方式對 component 測試？有沒有 API Documentation？如果都沒有肯定是混亂而已⋯⋯那還不如用 headless ui ⋯⋯

鑑別團隊架構能力也可以問打包方式，有沒有拆分功能型 package？用什麼打包？版本控管方式？

鑑別團隊流程能力可以問 CI/CD 流程，CI 什麼時候觸發？檢查哪些項目？CD 什麼時候觸發？有哪些流程？

鑑別團隊規範也可以問蠻多的。ESLint 會採用哪些 plugin？團隊規範會使用 Linter rule 來規範還是只依賴文件？PR和 Releases 流程？有執行 Code Review 嗎？通常 Code Review 看什麼？如果很趕，Code Review 會有什麼調整？

其實隨便一說就很多，通常你把自己對技術的渴望問出來就會問出很多。如果對方剛好是很優秀的公司，對於你問的問題深度能深深感受到你的程度，也可以變成很棒的加分項目。
