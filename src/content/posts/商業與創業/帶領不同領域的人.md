最近開始我的 Side Project，我的主題是做一個 UI Framework 。這並不是一件容易的事，不管是在「工程面」或是「設計面」都是門檻相當高的議題，但我覺得是件非常有趣的事。

除了我本來的專常外，我還得帶領多名設計師。起初找了約略7名 UI 設計師，我想從提案開始執行，這階段其實是在測試是否有意願。這時候許多人提出很多質疑，有人覺得不現實、有人覺得目標不明確、有人覺得無法協作。但事實證明，該陣亡的就會陣亡，只是想走馬看花的其實我也沒想要花時間在上面。

隨著我多次強調中心思想，大家慢慢理解我的目標，大家開始做了很多嘗試。我感覺到大家觸及到我的目標，我開始把主導權交付給具備統合能力的人，團隊慢慢開始獨自運轉起來。

我覺得放手交給團隊人員運轉很重要，過多的控制會使人感到疲倦。side project 陣亡率之高也不是什麼奇怪的事，又沒有強制力，初期也沒有成就感。不過我確實學到了很多領導上的經驗值，特別是領導一群比自己還專業的一群人，有很多技巧。
