前端常常在開發時，未必能保障後端或是外部資源都能是良好運作，很怕任何開發環境資源掛掉導致無法開發，又或是根本還沒完成開發就要先動工。這時 Mock Server 就很好用，模擬一個伺服器環境，通常可以用 nodejs 或 service worker 起一個仿後端的環境，模擬所有預想的行為。

當然也有一派人不認同這種作法，明明後端要開發一次，前端又要開發一次，無非就是拉高開發與維護成本。

你前端開發會起 mock 環境嗎？

還是會等後端寫完才動呢？
