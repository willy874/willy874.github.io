很多應用程式我們都會看到做 Web 和 App ，就算都是 Mobile ，但因為運行環境不一樣，所以操作也截然不同。本篇撇除工程意義上的不同，我直接來看對於使用者體驗上的不同。

到底有什麼不同？

Web 的操作行為不同，舉凡如上一頁、下一頁、重新整理、貼上網址、滑鼠右鍵、啟動閱讀器、修改放大比例⋯⋯諸多介面外的特殊操作。

Web 的權限不同，在 App 所有操作權限，舉凡定位、攝影機、檔案操作、通知系統⋯⋯都是直接收到作業系統管制，而Web 是基於瀏覽器之上再去做一次權限細分，而且全部是「預設關閉」，第一次請求會詢問開啟。只要任何一關檢查不過，通通無法操作。

Web 多了一個外框。因為多了一個追加功能的外框 UI，所以顯示空間更加擁擠，要更有效去管理介面上的顯示。

版本兼容問題，App 壽命直接受到「作業系統」限制，而作業系統受到「裝置」限制，加上行動裝置的壽命短，通常很難超過十年。但桌上型裝置動不動超過十年壽命，作業系統不升級更是常態，Web 要做向下兼容的範圍更廣、更難技術迭代。但相對的，強行為使用者升版也是 Web 的優勢，說升級就升級。

很明顯大致方向都是 Web 的限制更多，但操作方式更複雜，但因為限制和通用性，維護起來確實輕鬆不少。在諸多限制和基礎功能繁多的前提，其實 Web 很明顯是更難設計的，要考慮使用者會對你應用程式所做的操作其實更多。

最後我必須說，不要把 Web 當 App 去設計，你永遠不能去限制使用者行為，更無法阻擋使用者去做這些操作，不要異想天開去封鎖使用者行為，這些封鎖行為加諸的代價是需要更高昂的成本去維持的。
