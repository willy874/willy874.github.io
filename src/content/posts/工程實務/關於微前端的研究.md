在微前端的開發已經半年了，大部分時間都花在架構面和打包等等議題，也踩了很多很多的坑。但我希望可以更有系統整理這些資訊，我又重新讀了很多文章，然後嘗試各種 POC 實作。但不免跟使用的建構工具有點關係，因為微前端是靠網路傳輸來讀取跨應用的靜態 JavaScript 檔案。其中讓我踩坑踩到滿頭包到就是 vite，確實 vite 是劃時代的建構工具，打包的 config 介面也比 webpack 友好太多，但太多不成熟的成份在裡面，導致各種困難窒礙難行。

對於這個技術解決方案的熟悉度也因為近期的研究又提升不少，逐漸也發現各種新坑，但也因為近期的研究也抱持更多對於未來發展的期待。微服務 2014被提出來後蓬勃發展，而微前端是 2018年被提出來的，發展至今仍然沒有微服務來得普及，其中原因跟 web 的架構有很大的關聯，諸多難點和生態工具支持情況很差。加上前端技術近年爆炸成長，不管是 Bundler, SSR, SSI, Server Component 等等，微前端工具追不上生態圈工具的成長，微前端上手壁壘又非常高，能參與開發的人更是少之又少。

再不斷研究中我發現，想搞好微前端勢必要補一些跟網路、架構、系統相關的知識，它也屬於軟體開發元件的一個選項，所以如何整合也很重要。包含著 CDN, Server Render, Module Federation, Docker… 各種知識海量考驗著你。

近期應該會整理個大綱和簡報，想在社群辦個研討會來討論這個議題。

這些知識點留著後面一個個挑出來聊好了⋯⋯
