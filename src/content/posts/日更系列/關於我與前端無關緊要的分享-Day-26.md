對於 webpack, rollup, vite 你懂多少？

前端打包真的是一門超級難學，但又最不容易被看到的技能。

大部分的人學習打包都是止步於網路上部落格看到的知識點，或是能解決的簡單情境，看到有插件大概看一下文件就使用，基本上腦袋都沒有思考過在打包中的 pipeline 發生了什麼事情。所以大部分的人在打包工具出問題時，要不就繞路而行，要不就是直接放棄。你真的有認真閱讀過你包出來的 dist 長什麼樣子嗎？

近年 AI 興起，蠻多打包設定問題可以問問 AI，省去爬 Github issues 和 Stack overflow 的時間，但也不代表不需要懂那些知識。大部分的人遇到的問題都是爽快的把東西加上去，以為萬解。但如果不知道一些根本性的問題，其實你只是在疊加技術債，打包設定越多，增加太多你連做什麼都不知道的設定越多，要承擔未來的苦痛重量就越重。

最近都在處理跟打包的問題，突然間覺得好像使用比較新的技術在設定打包上越來越輕鬆，但坑卻埋藏的越來越深。有時真的想罵「這不是我知道的坑嗎？我怎麼忘記了」這種感覺。近期剛好有學生也問到了學習打包的問題，我也才在想我怎麼開始學習的？其實仔細想想我學會打包的過程還是挺地獄的，真的是在磨難中成長。

最近 UI Library Side Project 也處理了頗多打包問題，在一個個拿來分享吧。
