Portal 是許多元件的的基礎，也可以跟 Dialog 進行延伸與變化。舉凡 Select, Dropdown, Input Picker, Tooltip, Modal, Drawer 這些都是相關體系分類。他們的共同點就是脫離元素的地位，以視窗為主題來呈現資訊。

但正因為以視窗為主題，所以也存在更多的陷阱與處理上的困難點，許多邊際問題必須去處理。

點擊背景是否允許關閉展示？

比如超過視窗是否要限制？

內容過多該怎麼滾動？

如果非同步的資料進入如何正確展示？

彈出展示時，如果撞到邊界是否要換邊？

其實還很多⋯⋯

這部分的設計相信是很多元件設計者的煩惱，要處理到優雅、完善、彈性等等條件，真的不是一件容易事情。這幾天卡在內部 API 設計的思考上好幾天，一直未能有突破。

我現在稍微停下腳步去看看各家的元件怎麼實現，迷茫時不要瘋狂自幹，多去參考別人家的實作，有時候也是獲取靈感的方式。雖然不見得真的會喜歡，最少我可以很確定告訴別人每一家 UI Library 的實作細節。
