無論參考再多的開源庫，無論規避多少問題，技術選型永遠存在缺陷和困難。真的只有面對這些問題，去實際實現、體會、製作，慢慢得已感受每個細節的天工。

現在越來越熟悉 Chakra UI 生態系的底層實踐機制，我真的覺得它的抽象化完全是藝術，有很多設計精神非常之優秀。但如果單純使用它沒有什麼出奇之處，我覺得就不過是一個普通的 UI Library 。但它驚奇之處是在 Zag, Ark, Park 三套底層的抽象技巧，真的是非常有趣。有想要客製化開發 UI 的人，真的很值得學習的設計理念。

當然，我也是從模仿開始，進而發現問題、微調，最後意識到不簡單之處。慢慢也會踩坑，開始淬煉品味，看到各種複雜的問題。我非常想要分享，但太多問題的複雜度真的不是簡單可以用文字傳達的。
