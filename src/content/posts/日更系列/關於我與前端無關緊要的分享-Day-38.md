我朋友都知道我對於 React 不少怨言，但社群環境依舊是傾向著它，這讓我確信。市場需要的不是優雅的程式實作，名氣、認同、穩定，這些要素對於產品來說重要。連個軟體的 Library 都是這樣，更別說像樣的商業應用。

今早看到追蹤的工程師分享 React Conf 在探討 signals 在 react 的可能性，姑且不知道是否成為未來，也必須說除了 react 之外的所有框架都在走 signal 的心智模型。

坦白說我並不支持 React 第三方的 signal library ，因為這件事應該是響應底層的工作，透過第三方耦合的延伸問題其實蠻嚴重的。真的要做，還得痛定思痛去重構 React 舊有的程式碼。React 原始碼閱讀過都覺得很不舒服。依賴複雜，型別脆弱，解耦不良，對於 tree shaking 不友好。我不清楚 React 什麼時候願意改變去面對他的技術債，甚至某幾款「大家很愛的」的Library 也都是問題很大。

事實證明，成功的產品不需要多優秀的工程實作，不需要人見人愛的原始碼，它需要的是市場聲量和穩定。反正多數人從來不會去計較內部實作，他們要的就只有「可用可信賴的產品」。
