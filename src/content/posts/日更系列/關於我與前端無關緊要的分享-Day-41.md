UI Library 已經卡住兩週，都在調整架構。真的覺得 Primitive UI 的設計很困難，API 設計常常會卡到很多底層設計的盲點，東迴避西迴避。

剛好最近工作也需要重構 UI Library ，把工具鏈進行大幅度的升級，將較舊時代的技術轉移到這個世代的工具鏈。工具鏈一環綁著一環，要怎麼拆 Commit 和 PR 也是一個莫大的議題，我費了很大的勁。最後一整週來來回回都在測試，但總是有可能有些 Edge Case 藏匿在某些地方。如果整個技術線和方案沒辦法足夠真實的測試真實場景，那東西的穩固性極爲困難，其實有種經常在走鋼索的感覺。

我只能說，漸進式處理吧...

測試案例真的很困難，真的要寫，動不動都比原始碼大上 2~3 倍，維護 Test Case 難度又比原始碼困難... 測試真的是軟體難題，成本和穩定的天秤上膠著。
