現在幾乎壟斷前端市場的 tailwind ，語法幾乎成爲 Atomic CSS 的「標準」，在這之下去擠壓其他解決方案的空間。或許統一是好事，但社群發展因此就不鼓勵異聲，會得到「為什麼還要造新輪子？」這樣的質疑。

但 tailwind 是最好的方案嗎？

其實我也不知道，在設計 UI Library 中有種被 tailwind 綁架的感覺，但我覺得 tailwind 並沒有跟 UI Library 這種東西很合。他不能動態拆分 CSS Chunk、掃描原始碼需要被曝露出來、CSS 變數架構更加複雜而隱晦⋯⋯

這些問題依然是存在，並沒有因為演進消失。鼓吹著 zero runtime 的現在，被忽視的動態加載機制，這真的是趨勢？

我不覺得 tailwind 是唯一的選擇，過去很多做法也都有它的優勢與價值，重新審視架構去評估不同的解決方案的價值還是很重要的。
