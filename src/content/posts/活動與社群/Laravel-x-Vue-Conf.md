這次活動難得可以遇上前端圈的開源大大尤雨溪，可以聽他分享關於 Vue 發佈以來各種心酸血淚，從一個頂尖開源項目的主導者口中聽他分析的各種想法與策略。但我覺得聽到這些血淚史更重要的是從這些教訓中學習到對於開發的各種細節，從更高的層次去看待每一個專案的使用與升級。

1. 一個成功的開源套件或規範一定要去聽取社群的聲音，社群對於特定的方法而壯大一定有背景因素與價值。
2. 破壞性的改版，每一個點看起來都不是很嚴重的問題，但綜合起來卻是一個很大的橫溝。版本迭代過程再慢慢修正API，而不是一次做版本大更迭。
3. TypeScript 雖然無法成為規範，但如今已經成為事實規範了。團隊當到達一定的人數，使用 TypeScript 是對維護性的負責，它是有相當的好處。
4. 未來 Vue 生態圈會減少對 API 的改動，漸漸往底層去優化，讓效能、效率、開發者體驗再進一步提升。
5. 插件的開發要避免對私有 API 對使用，對於一個套件的開發來說，原作者很有可能去經常更動私有 API ，這是非常危險的。
6. Composition 這樣的設計模式是非常適合 TypeScript 開發，也很便於管理大型的業務應用場景。因為 class API 太依賴裝飾器，去依賴一個還不穩定的標準不是一件好事，當初放棄 class API 的設計是正確的選擇。
7. 將核心功能的每一個部分拆分開，不但提高模組的管理效率，也更好的為 SSR 打下基礎。
8. 避免使用 polyfill ，讓源代碼更簡潔，並努力推動有價值的提案規範化。未來可以更進一步去簡化不必要的代碼。

其實還是有蠻值得談的內容，網路上也有大神分享逐字稿，推薦可以去看看。
