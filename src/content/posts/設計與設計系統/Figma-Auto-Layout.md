我覺得 auto layout 某種層面就模擬了 CSS Flex box 的特性，真的大幅改變了 UI 設計稿的生態。

在過去畫設計稿，大家都以為是畫圖，大家常用 Illustrator, Photoshop 來畫出介面，甚至有的直接給你 PDF或 JPG 等等。但身為前端工程師的我體驗到那個世代各種恐怖的溝通地獄，和設計師吵不完的架，永遠雙方僵持不下。

有了 Auto Layout 的世界，這條溝通橫溝被打破，有了一個可以被溝通的語言。雖然還是有些詞彙不相同，但終於有了可以被對等交談方案，設計師開始理解「網頁佈局」這件事。

剛開始我聽到的是⋯⋯

「Auto Layout 太花時間了，不可能都用啦！」

「為了效率，我覺得我部分使用 Auto Layout。」

很有趣的，現在設計師圈之中出現了歧視鏈。

「蛤？為什麼這個沒有用 Auto Layout 包！」

「Auto Layout 很麻煩嗎？那是你懶」

看著這個現象的變遷，我真的覺得很有趣⋯⋯

前幾年因為某間公司不重視設計，身為前端的我強迫要學習 Figma，因為他們認為「畫面都是前端的事」。當時惡補了 Figma，規劃了整個 UX Flow，希望可以為產品提供價值。到現在這些都是我和設計師可以平行溝通的語言，雖然時間有限，但我持續吸收關於設計的知識。

大家都會要求設計師學一些工程，也會要求設計師做點專案管理和產品規劃。然而，為什麼從來沒聽過有人要求工程師去學點設計呢？設計價值真的這麼不值嗎？我覺得這些都是藐視產品價值，設計價值也是最大體現前端價值的地方。不是什麼單純讓介面變漂亮，而是要創造更好的體驗其實要仰賴設計與前端。
