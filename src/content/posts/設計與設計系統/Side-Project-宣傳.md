因為沒有留言和愛心去頂流量，其實觸及真的蠻有限，但仍然有許多人願意私訊我與我聊聊，希望能再進一步擴大這個觸及率。

歡迎有興趣的幫我留言 +1

這個計畫你可以當作從頭建置一個 antd, mui, vuetify, element-ui 這樣的 UI Lirary。

其中我並不會基於什麼 UI 框架去打造，而是「造輪子」從頭打造，從基礎設施自定義 bundle script，到每個元件細節的功能完成都嘗試去製作。

我們會與 UI 設計師配合，依照他們所製作的Design Guideline 準則進行開發，不是單純只給工程師用，而是包含設計細節都得考慮。包含 i18n, Dark mode, container RWD, theme system, CSS inject等等考慮各種應用場景並實現。

希望做出可以通用在 Server side render 和 Micro frontend 兩種架構下都可以運行的 UI 框架。如果心有餘力我希望把核心邏輯抽出讓跨框架也可以去使用，而非單純整個綁死在特定框架邏輯上。

初版會先以 React 架構作為主體，如果你是擅長 Vue 的也可以協助建構相關基礎建設功能。

我這邊目前會使用到的技術包含：

TypeScript, rollup, webpack , storybook, monorepo … etc.

團隊氣氛很隨性，有空做，沒空講一下也沒人逼你。不會派任務，自己找自己有興趣的部分提出設計，然後實作。每兩週的週日晚上會進行會議同步大家的情況，像是分享會一樣一起討論一起分享專案與工作上的點點滴滴。

我這邊不希望有「人家做得很好了，我們做不贏」的心態，而是「人家做得很好，但我們要研究模仿後比他做的更好」。目前我只考慮使用通用的基礎套件如 dayjs, lodash 這類型的，而非使用現成解決方案。我希望集結眾人力量去把每個細節打造的更好！
