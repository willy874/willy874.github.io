這是一個前陣子很紅，但有點過時的問題。但最近發現又有點像月經文，好幾次都有人提出來問，不免又在意起來。

原問題如下：

目前市場上有：Gdaddy, Wix, Shopify等很多版型網站，都可以自行購買域名串接，bk, bt,banner 那些位置動畫都可以讓美編自己改，網站也會針對PC, pad ,mobile等不同端用戶自動調整UI介面，收費也不貴，等於老闆只要請個會套版修圖的美編即可。

那為什麼老闆還要請前端？

這問題看似跟 AI 扯不上關聯，但我覺得也有關係，一樣可以回到一樣的問題，「前端會不會被取代」。

坦白講，這個情境只能用在「官網」或是商業邏輯單純的網站，沒有複雜的資料互動，沒有絢麗的動畫效果，沒有客製化的使用者體驗規劃。所以我認同，這種簡單的網站根本不需要前端。

但如果你要極端優化使用者體驗，要處理很複雜的商業邏輯，甚至是功能性非常特殊的網站，恐怕 No Code 工具是滿足不了的。

舉幾個例子，這種工具做得了一個繪圖編輯器？ 這種工具做得了一個 ERP 系統？ 這種工具做得了一個聊天軟體？ 這工具做得了一個交易所？

很顯然答案很明顯。

你可能心想，啊這些網站那麼複雜，有幾個前端工程師信心滿滿說我做得出來？很明顯，它需要很多專業人員協作完成，也需要一定程度的專業知識支撐。

可想而知，最慘的就是底層的工程師，隨時可能被這些 No Code 工具和AI淘汰⋯⋯
