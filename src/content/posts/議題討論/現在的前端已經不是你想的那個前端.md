或許有些公司還在用舊技術

或許有些公司還在努力轉型

最近不少朋友來諮詢我關於前端架構的設計

在溝通過程中最要命的問題是「團隊或主管對前端的不理解」

明明已經是成熟的軟體產品

功能大巨大到跟巨獸一樣

但對前端的認知還在「寫寫HTML檔案加上一些CSS和JavaScript就可以出現的技術」

或許你的產品不需要這麼複雜的前端

但如果是需要的

那請改變一下認知了⋯⋯

以我來說，我自稱「純前端」

對，我一直做前端，我也只關心前端

曾經有人跟我說「前端沒什麼好學的」，現在我只能黑人問號❓

我怎麼覺得我還沒學完？？？

經常舊時代的軟體工程師覺得

必須要懂 Web Service, Database, Performance, Cloud Service, Micro Service, Infrastructure, System Design, Container Management …

前端都不用懂，弄好畫面就好，太簡單了！

前端要學 Web Service ，現在的 SSR 就是一個 Service，它不再是靜態檔案。

前端要學 Micro Service ，只是我們關注的是 Micro Frontend 的 Service 應用。

前端要關注 Performance ，但它關注的是 Rendering performance 。

前端一樣要學 Deployment ，如果不懂的前端，怎麼會覺得SRE會懂怎麼部署？以為扔扔靜態檔案就是前端部署？還有容器開發和容器管理、CI/CD⋯⋯

前端一樣要學 Database ，在瀏覽器上一樣能玩SQL Lite, IndexDB…

除了上述這些，還有各種知識點⋯⋯⋯

瀏覽器和設備特性差異，這個排列組合遠遠比在伺服器的差異更多也更不可控。

使用者體驗並不是單純UI/UX 的事，前端經常也會需要懂，處理很多程式測試測不出來的微小感受。

動畫處理的非同步和極端的渲染議題，只要有一定規模都一定會遇上渲染難題，如何減輕使用者設備負擔。

打包議題，比 Compiler 更麻煩的問題，複雜的工具鏈和生態系，只為了盡可能縮小下載的程式總體積。

真的是⋯⋯求不要更新，學不動了⋯⋯

你還想到什麼我沒提到的嗎？

或是有什麼鬼故事能與我分享呢？
