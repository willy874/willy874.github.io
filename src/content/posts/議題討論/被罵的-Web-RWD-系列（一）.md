有鑑於這個系列三觀跟很多人不合，也會不小心噴到很多設計師工程師，我就直接歸類為被罵系列。我覺得這個應該也可以開很多集。

======

對於用戶來說。電腦板不單單只是閱讀空間比較大，甚至很多螢幕會再次進行放大，所以文字大部分情況在電腦板的都沒什麼問題。

以按鈕為例，依照我的開發經驗，最適合閱讀和操作的一個組合。

FontSize 18px

LienHeight 1.5

Padding 8px 16px

很大？確實習慣來說確實蠻大的。

但到了「手機版」這種小螢幕場景，很多設計師習慣性直接把文字大小壓低到 14px 甚至 12px，縮減各種間距，極盡所能壓榨出更多畫面空間。整個畫面極其擁擠，甚至大幅度改變排版邏輯，只為了畫面可以更多東西。

但我也知道，很多設計師有苦說不出口。業主希望「可以一個畫面看到全部東西」，但這件事對於使用者來說本身就是負擔。手機已經夠小夠難閱讀了，所有的元素又要再縮小一號。桌機明明可以足量留白產生出足夠的美感，降低資訊量，但卻瘋狂塞入不必要的資訊讓閱讀者更加疲累。

雖然像是 Bootstrap 很早就支援 RWD 字體，但其實根本滿足不了現實情境。最終前端只好手動追加大量的 media query ，sm, xs, xxs, xxxs…

最可怕的是「年長者必殺技」，字體放大 200%。然後發現很多按鈕看不到字，甚至破版到連點擊都點不到。網頁不是純粹服務「好看」，而是拿來「使用」，如果為了好看而失去彈性的易用性就本末導致了。不要老是嘲笑長輩不愛用 3C，是整個軟體設計根本對這些「長輩」和「視覺障礙者」非常不友好。

也有人說，年輕人好像沒差？確實沒差，文字小或大，年輕人實在不在意，但對於眼睛不好的族群真的是折磨。那你的 TA 是誰？難道你想跟我說「都可以是 TA」？我怎麼覺得這句話很眼熟？

我經常說的，UI 設計不只是畫圖的。我聽過一句話：「UI/UX 本身就是一體兩面，UI 只是 UX 的一種體現」，你到底是在做 Web 還是做海報呢？
